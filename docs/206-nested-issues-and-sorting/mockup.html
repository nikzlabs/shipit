Nested issues + two-level sorting

Interactive prototype for the ShipIt Issues panel. Sub-issues render nested under their parent; change the Sort by … then by … controls to re-order. Children are sorted within their parent only — they're never lifted into the top-level order (no Linear-style duplication). Toggle the disclosure triangles to collapse a parent.

What this prototype demonstrates
  • Nesting: a parent shows a disclosure triangle and a child-count pill (3). Children indent under it with a tree elbow. Collapsing hides the subtree.
  • Two-level sort: the top-level list is ordered by the primary key, ties broken by the secondary key, with identifier as the final tiebreak. Each axis has its own direction toggle.
  • Children stay home: sub-issues are sorted by the same comparator within their parent and never appear at the top level — unlike Linear, which lists a sub-issue both under its parent and again in the flat order.
  • Orphan handling: a child whose parent isn't in the current set (e.g. parent is Done and hidden) is promoted to the top level with a ↳ in SHI-300 hint, so it's never silently dropped.
  • Group-by (optional): toggle Group by primary to render the primary key as sticky section headers instead of a sort axis — parents still nest inside each section.
