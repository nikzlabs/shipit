Inline single-issue detail view in the Issues tab

Nik Zherebtsov
design issues frontend

Reading an issue should never require leaving ShipIt. Today every per-issue link — list rows, the agent's read/write cards — bounces the user to Linear or GitHub.

Goal

A master-detail view inside the Issues tab. The list rows and the agent's chat cards open it; the deep link to the tracker lives only here, in the header.

  • List row → detail (seeded, instant paint)
  • IssueRefCard / IssueWriteCard → detail
  • External link is the one escape hatch
