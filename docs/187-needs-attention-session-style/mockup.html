Needs-attention — every option, one screen

All variants explored, same 4-row list each: normal · two adjacent attention rows (to show stacking) · a selected + attention row (bottom, gray fill — the collision case). Light theme. Amber = --color-attention. The four families differ mostly in weight and in implementation cost:

Row borderAn accent on the row edge(s). Reuses the existing token in every theme.no new tokens
Row backgroundTints the whole row — strongest, but collides with the selected fill, so needs a both-state rule.+collision rule
Title fillSolid color behind the title, white text. Boldest. Needs a fill+text token pair per theme.+2 tokens × 14 themes
Title outlineBorder around the title only; text/bg untouched. Quiet & cheap.no new tokens
