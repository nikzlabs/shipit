Issue chips in the PR card

Folded into the existing collapsible “changed docs” panel (docs/205) — no new surface. Issues are deduped from two sources and shown as clickable chips that open the issue inline.

Sources merged & deduped: ① first user message & session origin (created from an issue) ② PR body Closes / Refs pointers Trackers: Linear & GitHub
1

Inline, issues first, divider

Issue chips lead the same wrapping row, separated from file chips by a thin divider. One row, minimal new concept.

Egress: durable allowlist store + containment Settings UI +612−88 ✓ CI 4/4 ▤ ⌄ ⋯
SHI-90Durable egress allowlist #1366Non-root worktree handoff egress allowlist shipit.yaml

+ Fewest moving parts; reuses the row verbatim.   − With many files, issues can wrap away from the top and lose prominence.

2

Labeled sub-rows

The panel splits into two labeled mini-sections. Issues always sit on their own line above the files, so they never get buried.

Egress: durable allowlist store + containment Settings UI +612−88 ✓ CI 4/4 ▤ ⌄ ⋯
Closes
SHI-90Durable egress allowlist SHI-145Non-root setup
Docs
egress allowlist shipit.yaml

+ Issues stay prominent; labels read as plain English.   − Taller panel; the “Closes” label folds intent into the section instead of the chip.

3

Intent-led pill, single row

Each chip carries its own verb segment (Closes / Refs / From session), so source & intent travel with the chip and order doesn’t matter. Files follow after a divider.

Egress: durable allowlist store + containment Settings UI +612−88 ✓ CI 4/4 ▤ ⌄ ⋯
Closes SHI-90 Refs #1366 From session SHI-201 egress allowlist

+ Self-describing; surfaces the session-origin issue even before any PR body exists.   − Verb text widens chips; titles dropped to keep them compact (full title in tooltip).

4

One issue chip in the header + full list in panel

The single primary issue (the Closes, or the session origin) rides in the always-visible header next to the title; the rest live in the expandable panel. Glanceable without expanding.

SHI-90 Egress: durable allowlist store + containment Settings UI +612−88 ▤ ⌄ ⋯
Also refs
#1366Non-root worktree handoff
Docs
egress allowlist

+ Primary issue visible with the panel collapsed — zero clicks.   − Crowds an already-busy header; only works when there’s one clear “primary”.

All variants share one chip component & one deduped data source; they differ only in placement and whether intent rides in the chip or a section label. My pick: Option 3 (self-describing, covers the pre-PR session-origin case) — or Option 1 if we want the smallest change.
