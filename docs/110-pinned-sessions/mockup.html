Pinned sessions — sidebar mock

Faithful to the real SessionSidebar using ShipIt's dark-theme tokens. Pins are per repository: each repo group owns its own pinned set.

  1. Pinned sub-section sits directly below the repo's New session button — a thin Pinned · N header in the same style as the existing Recently resolved divider. Hidden when a repo has zero pins.
  2. Drag handle (left gutter, hover-revealed) reorders rows within the pinned set, reusing the native HTML5 drag pattern already used for repo reordering — the green line is the drop indicator.
  3. Filled pin glyph marks a pinned row, tinted with the accent so a pin is legible at a glance.
  4. Overflow → "Pin to top" on an unpinned row (pinned rows show Unpin instead). No cap on how many you pin.
  5. Pinning doesn't suppress signaling — a pinned or unpinned row still gets the amber needs-attention edge (docs/187) on CI failure or conflicts.
  6. Per repo — a second repo keeps its own independent pin; there is no global pinned list.
Ordering within a repo group: Pinned (by pinnedAt desc, drag-reorderable) → Active (recency) → Recently resolved. An explicit pin outranks the auto-demotion of a merged/closed session.
