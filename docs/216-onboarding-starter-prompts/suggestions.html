Starter-prompt suggestions

Candidate chips for the empty-session launchpad — regular repo sessions only. Pick what you like; I haven't touched the UI yet.

The bar each one has to clear: does it teach a ShipIt-specific thing a new user wouldn't think to ask for — and does it work as the very first message in a session?

Scope from your feedback: sandbox sessions get no launchpad (power users). Dropped “Open a PR” (automatic, not a discovery) and “Write tests” (not ShipIt-specific).

✅ Recommended set · 4 · my pick

Second opinion from another agent

Recommend
🤖 Second opinion from another agent

“Ask Codex for a second opinion — have it review this codebase and flag anything risky or worth refactoring.”

Cross-agent review is genuinely unique to ShipIt and almost nobody knows it exists — the session agent consults a different model inline, in the same turn, and returns its findings to the chat.

README: “Cross-agent second opinions” (Review & ship)

What should I work on?

Recommend
🧭 What should I work on?

“Show me the open issues for this project and suggest which one to start with.”

Surfaces the brokered, tracker-neutral issue access — the agent reads Linear/GitHub issues without tracker tokens ever entering the container. Most users never realize the agent can see their tracker.

README: “Agent issue access”, “Inline Issues tab” (Plan & track)

Diagram this codebase

Recommend
🌳 Diagram this codebase

“Draw a diagram of how this codebase is structured and how the main pieces fit together.”

Teaches the Present tab — the agent renders a real diagram inline, not just prose. Very non-obvious, and a satisfying first “wow”.

Capability: the present tool (not in the README feature list)

Report a ShipIt bug

Recommend
🛟 Report a ShipIt bug

“Something in ShipIt isn't working right — help me put together a bug report for the team.”

The one you flagged: file a redacted, consent-gated ShipIt bug straight from chat. Works at any time, and turns a frustrated moment into a filed issue.

README: “bug-report secret redaction” (Security)

👁️ See it running

“Set up a live preview so I can see this app running as we work.”

Compose-native preview is a headline feature — but on a repo that already declares Compose it just runs, so the prompt only lands for repos without a preview yet.

Parallel PR-shaped sessions are distinctive, but this needs a concrete task in mind, so it's a weaker first message.

Explain this project

Maybe
📖 Explain this project

“Explain what this project does and how the codebase is structured.”

The most natural first action in an unfamiliar repo — but it's a generic LLM ability, not ShipIt-specific. Great onboarding, weak on the “ShipIt-specific” axis. Best candidate for a 5th, soft on-ramp.

Capability: generic

❌ Ruled out · with reasons

✕ Open a PR
ShipIt auto-commits every turn and opens the PR for you — automatic, not a discovery.
✕ Write tests for weak spots
Useful, but a generic coding ability — nothing ShipIt-specific to discover.
✕ Deploy to Vercel / Cloudflare
Deploys are triggered by the user from the UI, not the agent — so it can't be a chat prompt at all. (deployment.md)
✕ Cut a release
A real ShipIt capability, but never a first-message task — it presupposes shippable work.
