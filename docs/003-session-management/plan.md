# Session Management

## Two Session IDs

- **App session ID**: `crypto.randomUUID()` generated by the server on session creation. Stable identifier for workspace directories, metadata, chat history, and usage tracking.
- **Agent session ID**: Claude CLI's `session_id` from `system.init` event. Stored as `SessionInfo.agentSessionId`, used for `--resume` on subsequent turns.

## Session Isolation

Each session gets its own workspace directory with its own git repo.

### Directory layout

```
/workspace/
  sessions/
    abc123/          ← Session A's workspace
      .git/
      package.json
      src/
    def456/          ← Session B's workspace
      .git/
      index.html
  .vibe-sessions.json
  .vibe-chat-history/
  .shipit-usage.json
  .shipit/
    system-prompt.md
```

### Lifecycle

1. **Creation**: On first `send_message` (without `sessionId`) or `apply_template`, the server generates a UUID, creates `/workspace/sessions/{uuid}/`, initializes git repo, configures GitHub credentials if available.
2. **Per-connection state**: Each WebSocket connection tracks `activeAppSessionId` and `activeSessionDir`. All file/git operations use these.
3. **Per-session GitManager**: `AppDeps.createGitManager(dir)` factory creates instances per session directory.
4. **Switching**: `get_chat_history` activates a session, updating `activeSessionDir` and restarting FileWatcher for the new directory.
5. **Archiving**: `archive_session` marks a session as archived (hidden from sidebar). All data — workspace directory, chat history, usage, threads — is preserved.
6. **Migration**: Sessions created before isolation (no `workspaceDir`) fall back to root `/workspace`.

### Per-connection helpers

- `getActiveDir()` — returns `activeSessionDir` or throws if no session active
- `getActiveGitManager()` — returns GitManager for the active session directory
- `createSessionDir(sessionId)` — creates directory, initializes git, returns path

## Session titles

Auto-generated from first message (first 80 chars). Renamable via `rename_session` (pencil icon → inline editor → Enter/blur to save, Escape to cancel).

## Key files

- `src/server/sessions.ts` — `SessionManager` class: track, get, list, delete, rename, setAgentSessionId
- `src/server/types.ts` — `SessionInfo` type (id, title, agentSessionId, workspaceDir, lastUsedAt)
- `src/server/index.ts` — `createSessionDir()`, per-connection state, session handlers
- `src/client/components/SessionSelector.tsx` — Session list, resume, new, delete, rename UI
- `src/client/App.tsx` — `localStorage` persistence of current session ID, session state
