Title highlight — how it works across themes

The fill + text colors can't be one constant: a light surface needs a dark fill + white text, a dark surface needs a bright fill + dark text. So instead of hardcoding amber-800, add a two-token pair that each theme sets — --color-attention-fill and --color-attention-on-fill — exactly like the existing single --color-attention already differs per theme.

Why the text color flips (not just the fill)

Fix CI on auth refresh✓ dark theme — bright fill + dark text Fix CI on auth refresh✗ dark theme — light-theme fill barely separates from #030712

On the near-black dark surface, the light-theme's dark amber-800 pill (right) almost disappears; the bright amber-500 + dark text (left) pops. That's the whole reason it's a per-theme token pair rather than a constant.

Net: the border options need zero new tokens — they reuse --color-attention as-is across all 14 themes. The title-fill option is nicer visually but costs two new tokens × 14 theme files: light themes get amber-800 + white, dark themes get amber-500 + near-black. high-contrast can push further (pure #000/#fff); solarized can use its own amber. The component code is identical for every theme — it just reads the two vars.
