Notes. Sandbox accent = teal (distinct from PR violet, ops amber, folder yellow). The + lives above the session list (not in Settings) so every “advanced” session — Sandbox & Ops — is created from one discoverable place with its capability docs inline. Capabilities are independent, per-session choices — Git and Docker can each be on or off. The header chips reflect what this session was granted at creation (teal = granted, dashed/struck = not granted) and are immutable. This example is Git-only, which is why it clones & opens PRs but uses no Docker. Preview & PR tabs are removed (shown struck-through here only to illustrate the diff vs. a normal session). The in-session banner is human-facing orientation only; operational guidance for the agent (where to clone, use gh for PRs, pushed state is source of truth) lives in the system prompt, not the UI.
